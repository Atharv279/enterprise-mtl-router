Enterprise Auto-Routing Engine (Offline Multi-Task Learning Architecture)📌 Project OverviewThis project is a sophisticated, mathematically rigorous artificial intelligence pipeline designed for modernizing complaint management in high-security, offline environments. It replaces traditional keyword-based triage with a multi-modal ecosystem capable of autonomously ingesting multilingual text, high-fidelity audio, and video submissions without relying on external APIs.The system harmonizes unstructured inputs into a unified semantic representation and employs a Multi-Task Learning (MTL) framework to simultaneously classify incident priority and regress expected resolution times.🛠 Core Computational Architecture1. Multimodal Feature EngineeringLocalized Transcription: Utilizes faster-whisper (large-v3) with int8 quantization and Silero VAD to achieve high-speed, offline transcription while capping VRAM consumption at ~2.9GB.Visual Evidence Extraction: Implements PyScene Detect to isolate keyframes from video cuts and processes them through localized OCR to capture visible evidence like infrastructure IDs or printed warnings.2. Semantic RepresentationMultilingual Embeddings: Transforms aggregated text into a 1024-dimensional continuous vector space using the multilingual-e5-large-instruct model.Mean Pooling: Derives fixed-length sentence embeddings by applying a mean pooling operation over the last hidden state, incorporating an attention mask to neutralize padding tokens.3. Multi-Task Learning (MTL) EngineThe predictive core is a bespoke PyTorch network utilizing hard parameter sharing.Dynamic Loss Weighting: To prevent one task from dominating the gradient, the network implements homoscedastic task uncertainty. It learns observation noise parameters ($\sigma$) to dynamically balance the classification and regression losses:
$$\mathcal{L}_{total}=\frac{1}{2\sigma_{class}^{2}}\mathcal{L}_{class}+\frac{1}{2\sigma_{reg}^{2}}\mathcal{L}_{reg}+log(\sigma_{class})+log(\sigma_{reg})$$4. Algorithmic Routing & Capacity OptimizationVector Search: Employs LanceDB for zero-copy, embedded-first historical complaint retrieval via Approximate Nearest Neighbor (ANN) search.Global Optimization: Uses the Google OR-Tools CP-SAT solver to execute deterministic routing. This ensures assignments maximize semantic skill-alignment while strictly respecting predefined officer capacity thresholds.📊 Quantitative Performance MetricsThe model was trained on a synthetic dataset of 5,000 records and evaluated using scikit-learn:Metric TypeNameResultClassification (Priority)Accuracy0.8100Macro Precision0.8687Macro F1-Score0.6352Regression (ETA)Mean Absolute Error (MAE)4.8350 daysRoot Mean Squared Error (RMSE)6.4360 daysR-squared ($R^2$)0.5808🚀 Getting Started1.
